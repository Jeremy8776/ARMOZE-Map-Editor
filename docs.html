ARMOZE Desktop Guide

Current workflow for maps, zones, styling, extraction, and export

This documentation reflects the current Electron desktop app. Use it as the reference for the toolbar, map tabs, built-in extractor, floating zone styling panel, snapping, project save/load, and export flow.

Quick Start Tools Zone Styling Extractor Export Shortcuts

Quick Start

ARMOZE is now packaged as an Electron app. Start it with npm start, then either drag a map image into the editor or select an installed local map from the landing screen.

1. Open or import a map

Load a `.png`, `.jpg`, `.dds`, or `.edds` file, or pick a local map card from the built-in browser.

2. Draw your zones

Use the left toolbar to create rectangles, circles, lines, polygons, or freehand boundaries.

3. Style the selection

The floating properties panel lets you tune fill, border, pattern, label, and profile settings live.

4. Export or save

Save the project as JSON, or export scripts and overlays for Workbench or downstream tools.

Local map browsing, extractor output folder helpers, and map asset import are desktop-only conveniences. Core drawing and export logic still live in the renderer layer.

Workspace Model

The map view now behaves like a full-bleed workspace behind the toolbar, tabs, and layers panel. The UI chrome floats above the map instead of reserving a hard safe zone around it.

  • The left toolbar is an overlay for tools, zoom, snap, extractor, and documentation.
  • The right panel shows layers and a compact live data footer for the selected zone.
  • The top tab bar lets you work across multiple maps while preserving per-tab zoom and pan state.
  • The floating zone properties panel stays anchored to the currently selected zone.

Tools and Interaction

Select

Select zones, drag them, resize them from handles, and drag floating labels when applicable.

Pan

Use the dedicated pan tool, hold the middle mouse button, or use your wheel for zoom-driven navigation.

Rectangle / Circle / Line

Use geometric shapes for safe zones, lanes, exclusion areas, or objective boundaries.

Pen / Freehand

Build custom polygons or sketch organic boundaries for terrain-driven overlays.

Snapping is available through the toolbar grid button or the S shortcut. Snap feedback is visual, and zone edges can align against the active world grid.

Zone Styling Panel

The floating properties panel is the main styling surface. It is organized around profiles, color, and collapsed style accordions rather than an old fixed sidebar form.

Profiles

Save and reuse style presets. Unsaved edits fall back to the Custom profile state automatically.

Color system

Quick swatches, custom wheel pickers, and section-specific recent colors exist for zone, label text, and label background.

Border and pattern

Control border style, border width, opacity, pattern type, density, thickness, and angle where relevant.

Labels

Edit label text, font, size, emphasis, colors, background opacity, and integration modes.

  • Floating labels can be dragged inside or outside a shape.
  • Border label integration and pattern label integration live inside the label section.
  • Section summaries update as you change border, pattern, or label settings.

Layers and Live Data

The right panel shows the current zone stack and a compact data footer. Selection, visibility, delete, and property expansion all start from the layer list.

  • The selected layer highlights in the list.
  • The bottom data readout updates live while the selected zone moves or resizes.
  • Layer cards reflect the active profile name and visibility state.

Map Extractor and Import Flow

The extractor button in the toolbar opens the desktop extraction workflow for map assets. It is designed for local tactical content pipelines rather than browser-only usage.

  • Set scan, output, and tools directories.
  • Search for a resource or run the interactive extractor mode.
  • Convert extracted textures to a standard output format.
  • After a successful extraction, import the resulting map asset into the local map browser with a chosen map name.
The extractor workflow depends on desktop APIs exposed through `window.electronAPI`, so it is expected to behave differently from the pure renderer features.

Export Flow

The current export modal presents four direct output formats plus a coordinate foundation block.

Format Current label Use case
Enfusion script EnfusionScript (.c) Direct code export for script-driven zone logic.
Overlay image Image Overlay (.png) Transparent map overlays and downstream texture workflows.
Portable state JSON Config Share or re-edit exported zone state outside project save/load.
Workbench import Workbench Plugin (.c) Plugin-style script generation for Workbench-side creation.

The export modal also includes origin offsets, map scale, invert-Y, and a calibration entry point so exported coordinates match your world conventions.

Project Save and Restore

  • Save Project stores the editable session as JSON.
  • Load Project restores the zone set and editor state for that project file.
  • Tabs preserve per-map zoom and pan while the app remains open.
